Two-Minute Mental Model

The Structure
Partner → Integration → Property

Airnest Homes SL is the Mews partner. The partner builds an integration called Airnest Ops Connector. The integration gets one ClientToken. Each property (Bandama, Veintiuno, Masanet, Emete) generates its own AccessToken after we're certified — via the Mews Marketplace.

Auth — How It Works
No login, no OAuth

Every API call sends three values in the JSON body: ClientToken (integration identity), AccessToken (property identity), and Client (set to AirnestOpsConnector 1.0). Both tokens never expire — there's no refresh flow. This is simpler than typical API auth.

Three Data Channels
Pull · Push · Stream

Connector API = we call Mews on demand to pull data. Webhooks = Mews sends us a notification when something changes (2–5 min delay). WebSockets = a live real-time connection that streams events as they happen. We use all three.

How Mews Reviews
Use case first, endpoints second

Milan will ask what are you building and why before diving into technical details. He cares about the business purpose. Minimal scope = faster approval. We submitted a tight scope — each endpoint has a clear reason. Stick to that story.

Glossary

AccessToken
A property-specific credential that identifies one Mews property (e.g. Hotel Masanet). One per property. Airnest will have four — one per property. Never expires.
Atomize
Airnest's Revenue Management System (RMS). Acquired by Mews in 2022. Already live at Airnest — pushes optimized pricing into Mews automatically 24/7 via Autopilot.
ClientToken
The integration-level credential that identifies Airnest Ops Connector to Mews. One token for the whole integration (not per property). Issued by Mews after certification. Never expires.
Connector API
The Mews REST API used for data pulls. Every call is an HTTP POST with ClientToken + AccessToken + Client in the JSON body. Base URL: api.mews.com/api/connector/v1/
Cursor pagination
Mews returns large datasets in pages. Each page includes a cursor (a token pointing to the next page). You call again with that cursor to get the next batch. Our connector handles this automatically.
OAuth
A login-based auth method (like "Sign in with Google"). Mews private integrations do NOT use OAuth — they use the simpler ClientToken + AccessToken pair. No login flow required.
PII
Personally Identifiable Information. Guest names, emails, phone numbers, passport/ID numbers, booking amounts. Must be encrypted, retained no longer than necessary, and never shared outside the system.
Portfolio token
A special AccessToken that covers all properties in one Mews account under one credential. We selected "multi-property level" in the form — Milan can confirm if this is actually issued to private integrations.
Rate limit
Mews allows a maximum of 200 API requests per 30 seconds per AccessToken. Exceed this and requests are rejected (HTTP 429) until the window resets. Our sync stays well within it.
Scope
The specific list of endpoints and webhooks an integration is certified to use. Calling anything outside your certified scope is not allowed. Scope can be expanded later via an amendment.
SES Hospedajes
Spain's Ministry of Interior system for reporting hotel guest data to police within 24 hours of check-in. Our dashboard tracks online check-in completion partly to support this legal obligation.
ServiceId
Mews's internal ID for an accommodation service (e.g. "Standard Hotel Service" for a property). Most API endpoints require this as a filter. We discover it once via services/getAll and cache it.
Webhook
Mews sends an HTTP POST to our server when something changes (e.g. a reservation updates). Typical latency 2–5 minutes. Reliable delivery — Mews retries on failure. We subscribed to 5 event types.
WebSocket
A persistent real-time connection. Unlike webhooks (Mews pushes) and Connector API (we pull), WebSockets stream events within seconds. We subscribed to 3 event types for the live dashboard.
